## Functional Result

The project now demonstrates five judge-relevant flows:

1. Allowed delegation:
   - `doc-agent -> enterprise-data-agent`
   - enterprise data returned
   - report assembled
   - audit written
2. Denied delegation:
   - `web-search-agent -> enterprise-data-agent`
   - blocked with `AUTHZ_001`
   - denial audited
3. Timeout / degraded flow:
   - delegated token still issued legitimately
   - downstream agent simulates timeout with `AGENT_002`
   - `doc-agent` returns degraded output instead of crashing
   - timeout and fallback are both audited
4. Unavailable-agent flow:
   - delegated token still issued legitimately
   - downstream agent rejects the task as unavailable with `AGENT_001`
   - upstream agent aborts report generation
   - both failure points are audited
5. Dynamic authorization denials:
   - task scope violation
   - off-hours violation
   - both are denied before token issuance and explain why in audit logs
